# Chainlit RAG Chatbot

A web-based question-answering chatbot built with Chainlit that uses Contextual Retrieval to answer questions about Chapter 8: Transformers from the Stanford NLP textbook.

## Features

- **Contextual Retrieval**: Uses enriched chunks with contextual information for improved relevance
- **Source Citations**: Displays the source chunks used to generate each answer
- **Interactive Chat**: Clean, user-friendly interface powered by Chainlit
- **Real-time Answers**: Fast responses using GPT-4o-mini and FAISS vector search

## Prerequisites

Before running the app, ensure you have:

1. **Completed Notebooks 01-03**: Run the following notebooks first to generate required data:
   - `01-document-processing.ipynb` - Extracts text and creates chunks
   - `02-naive-rag-implementation.ipynb` - Creates Naive RAG vector store
   - `03-contextual-retrieval-implementation.ipynb` - Creates contextualized vector store

2. **OpenAI API Key**: Get one from [OpenAI Platform](https://platform.openai.com/api-keys)

3. **Python 3.8+**: Ensure you have Python 3.8 or higher installed

## Installation

### 1. Navigate to the app directory

```bash
cd app
```

### 2. Install dependencies

```bash
# Create virtual environment (if not already created from project root)
python3 -m venv .venv

# Activate virtual environment
source .venv/bin/activate  # On macOS/Linux
# OR
.venv\Scripts\activate  # On Windows

# Install dependencies
pip install -r requirements.txt
```

### 3. Set up environment variables

Create a `.env` file in the `app/` directory:

```bash
cp .env.example .env
```

Then edit `.env` and add your OpenAI API key:

```
OPENAI_API_KEY=your-actual-api-key-here
```

Alternatively, you can export the environment variable directly:

```bash
export OPENAI_API_KEY='your-api-key-here'
```

## Running the App

### Start the Chainlit server

```bash
# Make sure you're in the app/ directory
cd app

# Activate virtual environment (if not already activated)
source .venv/bin/activate  # On macOS/Linux
# OR
.venv\Scripts\activate     # On Windows

# Set OpenAI API key (if not already set)
export OPENAI_API_KEY='your-key-here'

# Run Chainlit app
chainlit run app.py -w
```

The `-w` flag enables watch mode, which automatically reloads the app when you make changes.

### Access the app

Open your browser and navigate to:

```
http://localhost:8000
```

The exact URL will be displayed in the terminal when you start the app.

## Technical Details

### Models Used

- **Embedding Model**: OpenAI `text-embedding-3-small` (1536 dimensions)
- **Generation Model**: OpenAI `gpt-4o-mini`
- **Vector Database**: FAISS (IndexFlatL2)

### Configuration

Key parameters (defined in `app.py`):

- `TOP_K = 3` - Number of chunks to retrieve
- `TEMPERATURE = 0` - Deterministic generation
- `MAX_TOKENS = 500` - Maximum answer length

### Data Files Required

The app expects the following files to exist (generated by running notebooks 01-03):

```
../vectorstore/contextual_retrieval/index.faiss
../data/chapter8_contextualized_chunks.json
```

## Troubleshooting

### Error: "OPENAI_API_KEY not set"

**Solution**: Make sure you've set the environment variable:

```bash
export OPENAI_API_KEY='your-key-here'
```

### Error: "Vector store not found"

**Solution**: Run notebooks 01-03 first to generate the required data files:

```bash
# From the project root directory
source .venv/bin/activate  # Activate venv
jupyter notebook
# Then run notebooks 01, 02, and 03 in order
```

### Error: "ModuleNotFoundError: No module named 'chainlit'"

**Solution**: Install the required dependencies:

```bash
# Make sure venv is activated
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### Error: Virtual environment not found

**Solution**: Create the virtual environment first:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Port 8000 already in use

**Solution**: Specify a different port:

```bash
chainlit run app.py -w --port 8001
```
